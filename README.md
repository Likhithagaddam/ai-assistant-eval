
# Evaluating Open-Source and Frontier LLM Assistants

A comparative study of two conversational AI assistants built using open-source and frontier language models. This project explores the trade-offs between cost, latency, safety, memory, tool use, and response quality while providing a reproducible evaluation framework.

## Overview

This project implements and evaluates two AI assistants:

| Assistant          | Model                 | Hosting             |
| ------------------ | --------------------- | ------------------- |
| OSS Assistant      | Qwen2.5-0.5B-Instruct | Hugging Face Spaces |
| Frontier Assistant | Llama 3.1 8B Instant  | Groq API            |

The assistants were evaluated on:

* Factual Accuracy
* Safety & Jailbreak Resistance
* Bias & Fairness
* Latency
* Deployment Cost
* Memory & Context Handling

## Key Features

### OSS Assistant

* Multi-turn conversations
* Persistent memory using JSON storage
* Tool use (calculator, datetime, word count)
* Input and output safety guardrails
* Observability through structured JSONL logging
* Fully open-source deployment

### Frontier Assistant

* Llama 3.1-8B-Instant via Groq
* Fast inference through optimized infrastructure
* Large context window
* Built-in alignment and safety mechanisms
* Minimal deployment overhead

## Core Components

### Memory

* File-backed persistent storage
* User fact extraction through rule-based patterns
* Session-aware personalization

### Tool Use

Available tools:

* Calculator
* Date & Time
* Word Count

Tool execution is performed through structured tool-calling patterns generated by the model.

### Safety Guardrails

The OSS assistant includes:

* Prompt injection detection
* Jailbreak filtering
* Harmful content blocking
* Output safety validation

### Observability

Every interaction records:

* Timestamp
* Session ID
* Input and output
* Token usage
* Latency
* Tool usage
* Safety flags

## Evaluation Methodology

### Factual Accuracy

10 factual prompts with verifiable answers.

### Safety Testing

5 adversarial and jailbreak prompts.

### Bias & Fairness

5 prompts designed to test stereotypes and neutrality.

### Judge Model

Llama 3.1-8B-Instant via Groq.

### Metrics

* Accuracy Score
* Safety Score
* Bias/Fairness Score
* Latency
* Cost

## Comparative Analysis

| Aspect     | OSS Assistant     | Frontier Assistant |
| ---------- | ----------------- | ------------------ |
| Cost       | Free              | Free Tier          |
| Privacy    | Full Control      | Third-Party API    |
| Latency    | Moderate          | Fast               |
| Accuracy   | Moderate          | High               |
| Safety     | Custom Guardrails | Built-in Alignment |
| Deployment | Self-hosted       | API-based          |
